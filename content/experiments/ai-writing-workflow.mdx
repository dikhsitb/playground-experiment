## The Problem

Using AI for writing often produces generic output that sounds like everyone else's content. This workflow keeps the human in control at every stage.

## The Workflow

### Stage 1: Brain Dump (No AI)

Write a rough outline and key points in your own words. Messy is fine. The goal is to capture your actual thoughts before any model influences them.

### Stage 2: Structural Pass (AI)

Feed the outline to the model with this prompt:

```
I wrote this outline for a blog post. Don't rewrite it.
Instead:
- Identify gaps in logic or missing sections
- Suggest a better ordering if needed
- Flag any claims that need evidence

Outline:
[YOUR OUTLINE]
```

### Stage 3: Draft Expansion (Human + AI)

Expand each section yourself first. Then use AI only for:
- Tightening sentences you're unsure about
- Finding a better word for a specific phrase
- Checking technical accuracy

Never ask AI to "write this section."

## Key Insight

The model is most useful as an editor and critic, not a writer. Treat it like a sharp colleague who hasn't read your draft yet — not a ghostwriter.

## Tools Used

- Claude for structural analysis
- Local notes in Obsidian for the brain dump
- Final edit always done manually
